# GeeGoo API Routing

## Service Ports

| Service | Port | Base URL | Purpose |
| --- | ---: | --- | --- |
| MCP | `3120` | `config.geegoo_url`, same as `base_url` | Bot CRUD, quotes, workflows, reports, technical analysis, strategy APIs. |
| Signal | `3210` | `signal_base_url` | Signal and strategy generation APIs. |
| Analyze | `3230` | `analyze_base_url` | Analysis APIs. |
| Data | `3300` | `data_base_url` | Market data APIs. |

Legacy 5xxx endpoints and `mk-` API keys are retired. All local defaults generated by `geegoo setup` use the 3xxx service layout.

`mcp_token` must be read from config or environment; never hard-code it in workflows.

## Common MCP Calls

| Scenario | API |
| --- | --- |
| Execute pre-market workflow | `checkTradingDay` -> `getReportBotCodes` -> per-stock report steps |
| Query daily reports | `getStockDailyReports` |
| Technical analysis | `getMCPAnalysis` |
| Create pre-market report | `createPreMarketReport` |
| Query pre-market report list | `getPreMarketReports` |

## Capital APIs

| API | Parameters | Description |
| --- | --- | --- |
| `getCapitalFlow` | `period=DAY` | Capital flow. |
| `getCapitalDistribution` | `code` | T-1 capital distribution. |

## getMCPAnalysis Periods

| Use | Correct value |
| --- | --- |
| Hourly index analysis | `hourly` |
| Weekly stock support/resistance | `weekly` |
| Daily analysis | `daily` |

The response field is `analysis_result`.

## HTTP Contract

- Header: `Authorization: Bearer <sk-api-key>`
- Header: `Content-Type: application/json`
- `checkTradingDay` body: `{"mcp_token": "...", "code": "00700.HK"}`
- Recommended timeout: at least 60 seconds, with one retry for first-connection failures.
